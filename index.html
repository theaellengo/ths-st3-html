Abstract

Augmented Reality (AR) to bring virtual scenes to real life is becoming popular for use in engineering, education, arts and entertainment. On the other hand, Anamorphic illusions expand the possibilities for the emergence of unique designs in art and architecture by employing angle, depth, and perspective to communicate an accurate image. We present a mobile marker-based AR application that allows users to solve anamorphic puzzles. The system tracks a variable number of markers up to four at the same time, and renders a randomly generated anamorphic transformation of a 3D model at the center of the markers. Users can then move their camera to find the position at which the object looks whole and the puzzle is solved. An event system was used to facilitate behavior as the number of markers changed. The application however has a limitation where the systems involved in transforming the model do not work as expected the first time a set of markers is tracked.

Research Overview

There are existing markerless implementations of anamorphic puzzle systems which exist, such as AMON. However, unlike marker-based AR which detects and tracks the markers within the environment, markerless AR extracts environmental features to keep track of the virtual scene. Since anamorphosis involves many vector operations, if paired with environmental tracking, markerless implementations can be computationally expensive. Thus, there is an opportunity to address this problem through a marker-based implementation. This study designed a marker-based Augmented Reality system, in the form of a mobile application, which can dynamically alter the active area, allowing users to piece together n slices of 3D anamorphic puzzle illusions within a table-top setting. This was done through: (1) identifying the common camera perspectives within a tabletop setting, as well as scenarios in which markers usually get occluded or span outside the camera’s field of view; (2) exploring upon the different existing approaches that allow for extended tracking when the marker is outside the camera’s field of view; (3) designing an approach that can move and scale the virtual scene to preserve its anamorphic effect as markers are moved in the camera view; (4) evaluating the performance of the approach based on the usability of the system and the computational expense; and (5) testing the system’s performance with respect to an increasing number of markers and 3D puzzle pieces as they are anchored to the markers.

System Overview

The application tracks a variable number of markers and renders a corresponding game puzzle using anamorphic illusions. The application performs this through four primary modules: Marker Detection, Anamorphic Transformation, Multi-marker Behavior and Win-Condition Checking. An Event System handles communication with these different modules to reduce dependencies between modules. This framework is visualized in Figure 1. In Marker Detection, the application counts how many markers are visible to the camera, determines what specific markers these are, and determines their positions relative to one another. These information are necessary parameters for the other modules, and are broadcast to them through the event system.

In Anamorphic Transformation, pre-sliced models undergo transformations in scale and position to achieve an anamorphic effect. These transformations are referenced from a constant point, which serves as basis for the win condition. Transformations are also random and dynamic so that repeated playthroughs of the same level don’t look the same. In Multi-marker Behavior, the anamorphic sliced model produced by the previous module is positioned and scaled depending on the number and position of the markers. Slices are then distributed and parented to the appropriate markers.

In Win-Condition Checking, the application constantly checks whether the camera coincides with the solution point from which anamorphic transformation was referenced, provided a margin of error. When it is, an event is broadcast telling the other modules that the puzzle has been solved, completing a cycle of the application’s the gameplay loop.

System Features

1. Marker Detection

The developed application is able to detect markers and trigger the generation of virtual objects. The system recognizes the markers at any point during a game. The order and arrangement of the markers can be random, and the application should behave the same regardless of the markers used.

2. Anamorphic Transformation

The pre-sliced 3D objects generated by the system is able to transform in scale and position to achieve an anamorphic effect. There is a certain angle where the virtual objects are seen as part of a complete image when the camera is in this position. When viewed from other angles, the whole image may look distorted. The transformation is also random so that succeeding game-level iterations do not look the same.

3. Multi-marker Behavior

The system is able to perform the same whether one or multiple markers are present in the play area. A change from one-marker state to another state when new markers are introduced is possible in the application. The sliced anamorphic objects scale and redistribute according to the number of markers present in the play area.

  • Center Calulation
  • Model Rescaling
  • Slice Redistribution

4. Win-Condition Checking

A specific position of the camera, where the sliced virtual objects form a complete image, is identified as the solution to the puzzle. When the user reaches this position, the application shows a notification that the puzzle is complete and the user is able to win that level.
